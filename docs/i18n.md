# Интернационализация (i18n)

Локализация интерфейса построена на **i18next** + **react-i18next** с автоопределением
языка. Тексты push-уведомлений локализуются отдельно на стороне Cloud Functions —
см. [notifications.md](notifications.md).

## Поддерживаемые языки

- 🇬🇧 English (`en`) — язык по умолчанию (fallback).
- 🇺🇦 Українська (`uk`).

## Файлы

| Файл | Назначение |
|---|---|
| `src/i18n.js` | Инициализация i18next, ресурсы, детектор языка |
| `src/locales/en/translation.json` | Английские строки |
| `src/locales/uk/translation.json` | Украинские строки |
| `src/components/LanguageSelector.js` | Переключатель языка в UI |

## Структура ключей

Используется единый namespace `translation`; внутри него строки сгруппированы по разделам:

```
bubbles · mindmap · auth · common · theme · language · categories
```

Обращение к строке — по пути от раздела, например `t('bubbles.title')`.
Набор разделов в `en` и `uk` должен совпадать.

## Определение языка

`src/i18n.js` использует `i18next-browser-languagedetector`:

- порядок поиска: `localStorage → navigator → htmlTag → path → subdomain`;
- выбранный язык кешируется в `localStorage` под ключом `i18nextLng`;
- если язык не определён или не поддерживается — fallback на `en`;
- `debug` включён в `development`.

## Использование в компонентах

```javascript
import { useTranslation } from 'react-i18next';

function Example() {
  const { t } = useTranslation();
  return <Typography>{t('bubbles.title')}</Typography>;
}
```

## Как добавить язык

На примере немецкого (`de`):

1. Создайте `src/locales/de/translation.json`, скопировав структуру существующего файла, и переведите строки.
2. Зарегистрируйте ресурс в `src/i18n.js`:

   ```javascript
   import deTranslation from './locales/de/translation.json';
   const resources = {
     en: { translation: enTranslation },
     uk: { translation: ukTranslation },
     de: { translation: deTranslation }, // ← добавить
   };
   ```

3. Добавьте язык в список в `src/components/LanguageSelector.js` (код, название, флаг).
